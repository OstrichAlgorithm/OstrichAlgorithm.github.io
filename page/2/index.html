执行环境

执行环境定义了变量和函数有权访问其他变量。每个执行环境都有之与其对应的变量对象，环境中定义的所有变量和函数都保存于此。记得看过关于篇《悟透javascript》中所指javascript无形之处皆为面向对象的思想，应该就是这个原因了吧。

全局执行环境是最外一层的执行环境。以web浏览器为宿主的全局执行环境的变量对象是window

每个函数都有自己的执行环境，当执行到函数时，会进入环境栈，执行完后，弹栈，控制权返回到之前的执行环境。

音（Tone）

是一种物理现象。物体振动时产生音波，通过空气传到耳膜，经过大脑的反射被感知为声音。人所能听到的声音在每秒振动数为16-2000次左右，而使用到音乐中的音（不含音），一般只限于每秒振动27-4100次的范围内。音的高低、强弱、长短、音色取决于物体的振动。

ASMR,神奇的听觉效果

实在是惊奇地发现了ASMR（爱上某人）这个东西，痒痒的，很放松。

百度百科释义：

ASMR (Autonomous sensory meridian response)，中文译名“自发性知觉高潮反应”，或”颅内高潮“，是一种生理现象，通常表现为脑部非常舒服的麻木/刺痛的快感。通常由聆听悄声说话和角色扮演而引发,推荐使用耳机观赏，越好的耳机，体验效果越好。

来一段音频试试：

###为什么人听到耳边轻柔低语会觉得很舒服？

人对声音的感知，除了大家都很熟悉的声音三要素：响度、音调（基频）、音色（频谱结构）之外，还有声音的时间过程（声音的建立和衰减）和声音的空间属性等等。
轻柔的低语有着与正常语音不一样的谱结构，激发起人脑的认知反应；
人是可以通过双耳感受3D空间的声音的，所以我们能感受到声音出现在耳边或者身后；
除了人除了“听”声音还可以通过皮肤、耳部的绒毛感知声波和空气扰动；
相信很多人都对 “低语”、“耳边吹气”、“剃刀” 有不同的情愫，而人脑往往是综合多个感官形成了这种认知机制，无论是源自童年的声音记忆，敏感区，还是听到近处+后方声源而警惕的本能自我保护

###为什么通过使用耳机听ASMR类音频也能有这种体验？

网上流传的ASMR类音频大多是双耳录音（Binaural Recording）类的。双耳录音是使用两个麦克风，通过一定的制式，为听者重现一个3D立体声的声音效果。通常双耳录音会使用一种特制的人工头模型，或者由录音师佩戴特制的麦克风进行录音，所以双耳录音也被称为“人头录音”，除了使用上述两种方式也可通过后期处理实现类似的效果。双耳录音通常需要使用耳机重放。
为什么ASMR选择了双耳录音？双耳录音不同于其他录音制品，双耳录音在录制和重放过程中，会记录人的耳廓、耳道、头骨、肩部等对声波的折射、绕射和衍射。在声学上，我们用HRTF（Head Related Transfer Function），即“头部相关传输函数”来描述这种影响。正是由于HRTF的影响，人的大脑能根据经验判断出声音发出的方位和距离。
普通录音制品（房间立体声）通常是按照音箱听音方式制作的。通过耳机欣赏时，声音被耳机驱动单元直接“灌”进双耳，无法传递人的耳廓、头骨和肩部等对声音造成的影响，在这种情况下，人的大脑就无法准确判断声音的方位和距离。所以很多人觉得，用耳机欣赏音乐时，音场的感觉不自然。
而双耳录音制品（人头立体声）在制作时就保留了这些信息并传递给人脑，使听者可更为清晰的感受到声源的方位距离，甚至骨传导和气流的作用。目前，耳机设计者也越来越关注这种骨传导和气流对听感的影响，比如Apple EarPods 的 acoustic vents。
此外通过后期制作，也可产生声音定位在头中的效果，成为耳机的“头中效应”（In-Head Effect），对于普通录音制品的耳机重放，这可能是一个弊病，但对于双耳录音这也可以成为一种特殊音效。

Javascript

javascript

完整的javascript包括三个部分：

  • 核心（ECMAScript）
  • 文档对象模型（DOM）
  • 浏览器对象模型（BOM）

ECMAScript

根据ECMA-262定义的ECMAScript本身没有参照web浏览器标准。
ECMAScript定义的内容为：

  • 语法
  • 类型
  • 语句
  • 关键字
  • 保留字
  • 操作符
  • 对象

DOM

文档对象模型（DOM,Document Object Model）是针对XML拓展于HTML的API。Dom把整个文档映射一个多节点的结构。

  • 核心（DOM Core）
  • 标签（DOM HTML）
  • 视图（DOM Views）：不同文档的视图接口
  • 事件（DOM Events）：事件和事件处理接口
  • 样式（DOM Style）：css
  • 遍历和范围（DOM Traversal and Range）：遍历和操作文档树的接口
  • 保存和加载（DOM Save and Load）:统一加载保存
  • 验证（DOM Validation）：验证文档

BOM

浏览器对象模型（BOM,Browser Object Model）,给浏览器制作的接口，比如cookie,location,screen,window等
